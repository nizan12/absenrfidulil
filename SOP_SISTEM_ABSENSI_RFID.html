SOP Absensi RFID SMAIT Ulil Albab — Rev. 01
Bab 01

Pendahuluan & Tujuan

Latar belakang dan sasaran implementasi sistem

A Latar Belakang

Sistem Absensi RFID SMAIT Ulil Albab merupakan solusi digital pencatatan kehadiran siswa dan guru menggunakan teknologi Radio Frequency Identification (RFID) yang terintegrasi dengan platform web dan notifikasi WhatsApp real-time kepada orang tua/wali.


SOP ini disusun sebagai panduan operasional bagi seluruh personil yang terlibat dalam penggunaan sistem absensi, mulai dari registrasi data hingga monitoring kehadiran harian.

B Tujuan

  • Memastikan pencatatan kehadiran siswa dan guru dilakukan secara akurat dan otomatis
  • Menyediakan notifikasi real-time kepada orang tua melalui WhatsApp
  • Mempermudah proses rekapitulasi data kehadiran untuk pelaporan
  • Meningkatkan transparansi dan akuntabilitas kehadiran di lingkungan sekolah
  • Mendukung program digitalisasi sekolah sesuai visi SMAIT Ulil Albab

C Ruang Lingkup

SOP ini berlaku untuk seluruh pengguna sistem absensi RFID, meliputi: Super Admin, Kepala Sekolah, Staff Admin, Guru Piket, dan Operator.

Arsitektur Sistem

Komponen dan alur kerja sistem

A Komponen Sistem

KomponenKeterangan

ESP32 + MFRC522Perangkat pembaca RFID yang terpasang di gerbang/kelas. Terhubung WiFi.

Dashboard WebAntarmuka admin untuk kelola data, monitoring, dan rekapitulasi.
WhatsApp (Fonnte)Layanan notifikasi otomatis ke orang tua saat siswa tap masuk/keluar.

Catatan Penting

Sistem menggunakan logika alternating — tap pertama hari itu = MASUK, tap kedua = KELUAR, dan seterusnya bergantian. Setiap tap memiliki delay yang dapat dikonfigurasi per perangkat (default 5 menit / 300 detik) untuk mencegah double tap.

Akses Sistem

Dashboard: https://absen.smaituab-idl.sch.id
API Endpoint: https://absen.smaituab-idl.sch.id/api

SOP Registrasi Data

Menambahkan data siswa, guru, dan orang tua

A Registrasi Siswa

1

Login ke Dashboard

Buka absen.smaituab-idl.sch.id dan login dengan akun admin.

2

Buka Menu Data Siswa

Klik menu Data Siswa di sidebar navigasi.

3

Klik "Tambah Siswa"

Klik tombol + Tambah Siswa di pojok kanan atas.

4

Isi Data Siswa

Lengkapi: RFID UID (scan kartu via Mode 1 ESP32), NIS, Nama Lengkap, Kelas, dan Kategori (Reguler/Full Day/Boarding).

5

Simpan Data

Klik Simpan. Siswa siap melakukan tap absensi.

B Registrasi Guru

Prosedur sama dengan siswa melalui menu Data Guru. Data yang diperlukan:

FieldKeteranganWajib
RFID UIDDidapat dari scan kartu di ESP32 Mode 1Ya (Wajib)
NIPNomor Induk PegawaiYa (Wajib)
Nama GuruNama lengkap beserta gelarYa (Wajib)
No TeleponFormat: 08xxx atau 628xxxOpsional

C Registrasi Orang Tua

Data orang tua wajib diisi agar notifikasi WhatsApp dapat terkirim saat siswa tap.

Perhatian: Format Nomor HP

Pastikan nomor HP orang tua terisi dengan benar dan merupakan nomor WhatsApp aktif. Format: 08xxxxxxxxxx atau 628xxxxxxxxxx. Nomor yang salah akan menyebabkan notifikasi gagal terkirim.

SOP Import Data Massal

Bulk import melalui file Excel (XLSX)

A Prosedur Import

1

Download Template

Pada halaman terkait (Siswa/Guru/Admin), klik tombol Import lalu klik Download Template. File template Excel (.xlsx) akan terunduh.

2

Isi Data di Template

Buka file template menggunakan Microsoft Excel / Google Sheets. Isi data sesuai kolom yang disediakan. Jangan mengubah urutan kolom header.

3

Upload File

Kembali ke halaman import, klik area upload dan pilih file yang sudah diisi. Format: .xlsx atau .xls.

4

Proses Import

Klik tombol Import. Sistem akan memvalidasi dan memasukkan data. Notifikasi sukses/gagal akan muncul.

B Format Template per Modul

ModulKolom Template (Sheet 1)Sheet Referensi
SiswaRFID UID, NIS, Nama Siswa, Kelas, Kategori, Nama Ortu, No HP Ortu, HubunganDaftar Kelas, Daftar Kategori, Daftar Hubungan
GuruRFID UID, NIP, Nama Guru, No Telepon—
Admin/UserNama, Email, Password, Role, No TeleponDaftar Role
Template Multi-Sheet

Template Excel memiliki beberapa sheet. Sheet pertama untuk data import, sedangkan sheet lainnya berisi daftar referensi (kelas, kategori, hubungan, atau role) yang sudah ada di sistem.

Hal yang Harus Dihindari

– Jangan menghapus baris header (baris pertama)
– Jangan menambah/mengurangi kolom
– Hapus baris contoh/sample sebelum import data asli
– Pastikan RFID UID dan NIS/NIP tidak duplikat
– Kolom Hubungan: hanya isi ayah, ibu, atau wali
– Kolom Role: gunakan super_admin, kepala_sekolah, staff_admin, guru_piket, atau operator

SOP Proses Absensi Harian

Prosedur pencatatan kehadiran via tap RFID

A Persiapan Harian

  • Pastikan perangkat ESP32 menyala dan WiFi sekolah aktif
  • Cek indikator LED ESP32 — jika menyala stabil, perangkat siap
  • Pastikan jam server sudah sesuai (WIB / UTC+7)
  • Guru piket standby di area tap untuk membantu jika kartu bermasalah

B Prosedur Tap

1

Siswa/Guru Tempelkan Kartu

Tempelkan kartu RFID ke pembaca dengan jarak maksimal 3 cm. Tunggu hingga indikator menyala.

2

Sistem Memproses

ESP32 mengirim UID ke server. Server mencocokkan data dan mencatat waktu tap sebagai MASUK atau KELUAR.

3

Notifikasi Terkirim

Untuk siswa: notifikasi WhatsApp dikirim ke orang tua/wali yang mengaktifkan opsi notifikasi. Untuk guru: notifikasi dikirim ke Kepala Sekolah (hanya jika tap dari device bertipe classroom).

C Logika Tap IN/OUT

2KELUAR (OUT)"Ananda [Nama] telah KELUAR sekolah"

Siswa Boarding / Asrama

Siswa dengan kategori Boarding atau Asrama mendapat format notifikasi khusus dengan label Siswa Asrama. Dashboard juga mendukung Manual Tap dengan opsi paksa IN/OUT.

SOP Monitoring & Rekapitulasi

Pemantauan kehadiran real-time dan laporan

A Live Monitor

Menu Live Monitor menampilkan data tap secara real-time. Setiap kali siswa/guru melakukan tap, data langsung muncul di dashboard tanpa perlu refresh.

1

Buka Live Monitor

Navigasi ke menu Live Monitor di sidebar.

2

Pantau Data Real-Time

Data tap terbaru muncul secara otomatis. Informasi mencakup: nama, kelas, waktu, lokasi, dan tipe (IN/OUT).

B Rekapitulasi

Menu Rekapitulasi menyediakan laporan kehadiran berdasarkan rentang tanggal, kelas, atau individu.

  • Filter berdasarkan tanggal, kelas, kategori
  • Export data ke Excel untuk pelaporan
  • Lihat ringkasan statistik: hadir, terlambat, tidak hadir

Bab 08

SOP Notifikasi WhatsApp

Konfigurasi dan monitoring notifikasi Fonnte

A Setup Awal Fonnte

1

Daftar di Fonnte

Buka fonnte.com dan buat akun. Tambahkan device (nomor WhatsApp pengirim).

2

Connect Device

Scan QR Code dari dashboard Fonnte menggunakan WhatsApp di HP pengirim. Status harus berubah menjadi "Connected" (hijau).

3

Salin Token API

Klik tombol Token di dashboard Fonnte. Salin token yang muncul.

4

Masukkan Token ke Pengaturan

Buka menu Pengaturan di dashboard sekolah, paste token di field Fonnte API Token, lalu Simpan.

Status "Pending" di Fonnte

Jika pesan berstatus Pending, artinya device WhatsApp terputus. Solusi: buka dashboard Fonnte, klik Reconnect, lalu scan ulang QR Code.
